# Mobile Web Smoke Test

API 서버와 Expo Web을 실행한 뒤 브라우저에서 확인합니다.

```bash
API_HOST=0.0.0.0 API_PORT=3005 pnpm --filter @home-catcher/api dev
EXPO_PUBLIC_API_BASE_URL=http://127.0.0.1:3005/api/v1 pnpm --filter @home-catcher/mobile exec expo start --web --clear
```

| 항목 | 절차 | 기대 결과 |
| --- | --- | --- |
| 회원가입 | 새 이메일로 가입 | 홈캐처 세션 생성 후 가족 온보딩으로 이동 |
| 가족 생성 | `우리집 만들기`에서 가족명 입력 | 가족 생성 후 템플릿 선택 화면 표시 |
| 템플릿 적용 | 템플릿 목록/상세 확인 후 적용 | 홈 화면으로 이동, 템플릿 생필품/Todo/매뉴얼 생성 |
| 가족 초대 | 우리집 탭에서 가족 초대 코드 생성 | 초대 코드와 만료일 표시, 복사/공유 가능 |
| 생필품 재주문 | 생필품 목록 → 상세 → 재주문 확인 | 상품 정보와 구매 규칙 표시, 외부 상품 링크 열기 가능 |
| 구매 완료 | 구매 금액 입력 후 `주문했어요` | 구매 기록과 가계부 기록 안내, 중복 제출 방지 |
| Todo 완료 | Todo 목록 → 상세 → 완료 | 완료 기록 생성, 반복 Todo nextDueAt 갱신 |
| 매뉴얼 조회 | 우리집 매뉴얼 목록 → 상세 | 단계가 sortOrder 순서로 표시 |
| 캘린더 | 캘린더 탭 월간/일간 조회 | 일정, Todo, 고정지출, 생필품 소진 예상 표시 |
| 가족 일정 | 날짜 선택 후 일정 추가 | 월간/일간 달력에 새 일정 표시 |
| 고정지출 납부 | 캘린더의 고정지출 occurrence → 상세 → 납부 | dueDate 유지, 납부 후 PAID와 가계부 기록 생성 |
| 가계부 | 우리집 → 가계부 | 월간 요약, 내역, 직접 지출 입력 표시 |
| 가족 피드/고마워요 | 다른 구성원의 활동에 고마워요 | count 증가, 상대 계정 알림 생성 |
| 알림 읽음/보관 | 알림함 → 알림 상세 | 읽음 처리, 모두 읽음, 보관 후 목록 제거 |
| 가족 전환 | 우리집 탭에서 다른 가족 선택 | 이전 가족 데이터가 즉시 사라지고 새 가족 데이터 조회 |
| 로그아웃/세션 복원 | 로그아웃 후 재로그인, 브라우저 새로고침 | 보호 화면 접근 차단, 로그인 후 refresh 기반 세션 복원 |
| 서버 중단 | API 서버 중단 후 화면 새로고침 | 흰 화면 대신 오류/재시도 표시 |

실제 푸시 권한, DeviceToken, FCM/APNs 발송은 MVP Web smoke 대상이 아닙니다.
